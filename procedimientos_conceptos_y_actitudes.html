Procedimientos, conceptos y actitudes

Los procedimientos de sistemas operativos monousuarios incluyen características básicas como:

- Arranque del sistema, donde se carga el sistema operativo para que el equipo esté operativo para un solo usuario.
- Gestión de procesos que puede ser mono tarea (una tarea a la vez) o multitarea (varias tareas, pero un solo usuario).
- Administración de archivos y recursos del sistema, donde el usuario tiene todos los permisos y controles sobre los archivos y configuraciones del    sistema.
- Interacción con la interfaz de usuario, ya sea por línea de comandos o gráfica, para ejecutar programas y tareas.

Conceptos fundamentales:

1. Sistema operativo monousuario: Es un sistema diseñado para ser utilizado por un solo usuario a la vez. No permite accesos simultáneos de varios usuarios, sino que un único usuario controla el sistema durante su uso.
2. Monotarea o multitarea: Estos sistemas pueden ser monotarea (una tarea a la vez) o multitarea (varias tareas pero para un solo usuario), facilitando la ejecución de programas y procesos para un único usuario.
3. Facilidad y simplicidad: Son más sencillos de usar y administrar, ya que no manejan múltiples cuentas ni accesos simultáneos, lo que simplifica la gestión de recursos.
4. Acceso total: El usuario dispone de todos los recursos del sistema sin compartir ni restricciones de permisos multiusuario.

5. Limitaciones: No permiten la cooperación simultánea entre usuarios ni el manejo de múltiples sesiones, limitando su uso en entornos colaborativos o de red.

Actitudes

a) Responsabilidad en el manejo de los datos, ya que el usuario tiene control total sobre el sistema y puede afectar archivos e información vital.
b) Cuidado en la administración del sistema, dado que no existen roles diferenciados y un error puede afectar el funcionamiento general.
c) Uso consciente y ordenado de los recursos del sistema para evitar sobrecargas o pérdidas de información.
d) Mantenga hábitos de seguridad básicos, como evitar la introducción de software malicioso, dado que la protección multiusuario no está activada.
e) Vigilancia constante para prevenir errores o accesos no autorizados, ya que no hay múltiples niveles de control o restricciones de acceso.
